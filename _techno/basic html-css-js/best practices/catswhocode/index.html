Front End Best Practices

Indiquer quelle div se ferme

<div class="practice">
...
</div> <!-- .practice -->

Utiliser un reset CSS

Il existe pas mal de reset css sur internet.

  • Le plus simple : Eric Meyer
  • Un peu plus costaud : Normalize.css
  • Plus qu'un simple reset, un style de base : Knacss

Ne pas utiliser @import en CSS

L'utilisation de @import ralentit considérablement l'import des fichiers en comparaison à une inclusion normale avec link.

Optimiser les images

Utiliser un outil pour optimiser les images.

  • SmushIt
  • Package Gulp : gulp-image-optimization

Ne pas mélanger le CSS avec le HTML

La balise style est à proscrire !

Utiliser les commentaires conditionnel

<!--[if lte IE 6]>
<link href="ie.css" rel="stylesheet" type="text/css" />
<![endif]-->

PLacer le code javascript à la fin de la balise body

<script></script>
</body>

Tester la compatibilité cross-platform durant le développement

Des outils très pratiques existent pour ne pas installer tous types de navigateurs
Exemple : spoon.net
